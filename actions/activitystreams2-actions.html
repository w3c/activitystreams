This specification describes the Actions vocabulary.

Author's Note

This draft is heavily influenced by the JSON Activity Streams 1.0 specification originally co-authored by Martin Atkins, Will Norris, Chris Messina, Monica Wilkinson, Rob Dolin and James Snell. The author is very thankful for their significant contributions and gladly stands on their shoulders. Some portions of the original text of Activity Streams 1.0 are used in this document.

Introduction

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHALL NOT", "SHOULD", "SHOULD NOT", "RECOMMENDED", "MAY", and "OPTIONAL" in this document are to be interpreted as described in [[!RFC2119]].

Value Conventions

This specification uses IRIs [[!RFC3987]]. Every URI [[!RFC3986]] is also an IRI, so a URI may be used wherever an IRI is named. There are two special considerations: (1) when an IRI that is not also a URI is given for dereferencing, it MUST be mapped to a URI using the steps in Section 3.1 of [[!RFC3987]] and (2) when an IRI is serving as an "id" value, it MUST NOT be so mapped.

Notes:

An ActivityHandler that describes an HTTP request to load content within a Web Browser view. This handler type roughly corresponds to invoking a link or HTML form post. The method property indicates the HTTP method to be used. The method value MUST be either "GET" or "POST".

The browserContext property specifies the target Browsing Context Name as defined in [[HTML5]].

The sandbox property specifies the [[HTML5]] Sandboxing policy to be applied to the Browser Context.

The BrowserView class inherits the properties of the Link class. Accordingly, the href property specifies the target URL for the HTTP request. Alternatively, the hreftemplate property can be used to describe an [[!RFC6570]] URL Template an implementation can use to construct the target URL for the HTTP request.
Type Of: ActivityHandler AND Link
Properties:

Inherits all properties from ActivityHandler and Link.

Notes:

An ActivityHandler that describes embedded content used to carry out an Activity. This handler is intended to be roughly equivalent to Embedded Content in HTML5.

The MIME media type of the embedded content is specified using the mediaType property.

If an implementation chooses to display the embedded content within a browser context as defined by [[!HTML5]], the sandbox property specifies the Sandboxing policy to be applied to that browser context.

Determining the exact content to embed and how is largely dependent on the specified MIME media type that is specified. If the mediaType property specifies text or HTML based content (e.g. using any variant of "text/*", "application/html" or, "application/xhtml", etc), then the content to display MAY be specified using either the content or summary properties with content being the preferred means of specifying the full content to display and summary optionally providing a shortened or summarized version. The displayName and title properties can be used to provide captioning information for the embedded content. The content and summary properties MUST NOT be used for any type of content not intended to be directly viewable by a user either as plain text or processed as markup.

Alternatively, the url property can be used to provide the URL location where the embedded content can be retrieved.
Type Of: ActivityHandler AND Content

Notes: An ActivityHandler that describes an HTTP request/response flow used to carry out an Activity.

An ActivityHandler that describes an HTTP request/response used to carry out an Activity. This handler type roughly corresponds to using Javascript mechanisms within an HTML page to invoke an HTTP request. The method property indicates the HTTP method to be used. Unlike BrowserView, the method value can be any valid HTTP method.

The HttpRequest class inherits the properties of the Link class. Accordingly, the href property specifies the target URL for the HTTP request. Alternatively, the hreftemplate property can be used to describe an [[!RFC6570]] URL Template an implementation can use to construct the target URL for the HTTP request.

Unlike BrowserView, the HttpRequest handler is invoked independently of any user interface element. Implementations must determine on their own whether and how to process or display any payload returned in response to the HTTP request.
Type Of: ActivityHandler AND Link
Properties:

hreftemplate | method

Inherits all properties from ActivityHandler and Link.

Notes:

Describes one or more Activities that can be carried out with the containing object. The identification of the Activity can be qualified or unqualified as illustrated in the examples. In the unqualified form, the identifier of the Activity is provided directly and it is up to the implementation to determine exactly how the Activity is to be carried out. In the qualified form, the value is specified as an object whose @type provides the identity of the Activity type and the using property specifies one or more ActivityHandler instances that describe how the Activity can be carried out.

Implementations are free to ignore any Activity or ActivityHandler specified that is not supported by the implementation.
